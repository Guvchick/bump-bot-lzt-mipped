# Bump Bot (Lolzteam + Mipped)

Telegram-управляемый бот на Go, который автоматически поднимает («апает») ваши
темы на двух форумах:

- **Lolzteam** (`lolz.live`) — через официальный REST API по Bearer-токену.
- **Mipped** (`mipped.com`) — через сессию (логин/пароль) и штатную кнопку
  «Поднять тему» в рамках суточного лимита форума.

Управление — через inline-панель в Telegram. Доступ только для whitelisted
Telegram ID. Секреты (токены, пароли, cookies) хранятся в SQLite **в
зашифрованном виде (AES-256-GCM)**.

## Возможности

- 👤 Аккаунты: добавление Lolz (токен) и Mipped (логин/пароль), проверка
  авторизации, прокси (socks5), удаление.
- 📌 Темы: добавление по `thread_id` (Lolz) или URL (Mipped), ручной ап,
  вкл/выкл автоапа, настройка интервала, автоподтягивание заголовка.
- ⏱ Планировщик: per-thread расписание с `next_bump_at` в БД (переживает
  рестарт), джиттер, глобальная задержка между запросами, экспоненциальный
  backoff при ошибках, корректный разбор «подождите N часов» (RU/EN).
- 📊 Статистика: снимки просмотров/ответов, дельты за 24ч/7д, ASCII-спарклайн.
- 🧾 Логи апов и ⚙️ настройки прямо из чата.
- 🔔 Опциональные уведомления при `auth_failed` и серии неудачных апов.

## Стек

Go 1.26 · [`telebot.v4`](https://gopkg.in/telebot.v4) · `net/http` +
`cookiejar` · [`goquery`](https://github.com/PuerkitoBio/goquery) ·
[`modernc.org/sqlite`](https://modernc.org/sqlite) (чистый Go, без CGO) ·
`crypto/aes` (AES-GCM) · `log/slog`.

## Структура

```
cmd/bot/main.go         — точка входа, wiring, graceful shutdown
internal/config/        — загрузка и валидация .env
internal/crypto/        — AES-GCM шифрование секретов
internal/storage/       — интерфейс Storage + реализация SQLite
internal/forum/         — интерфейс Forum, клиенты lolz.go и mipped.go
internal/scheduler/     — циклы апа и сбора статистики, пейсинг, backoff
internal/tg/            — Telegram-панель: хендлеры, клавиатуры, FSM
migrations/             — встроенная (embed) SQL-схема, применяется на старте
```

## Быстрый старт

### 1. Получить данные

- **Telegram токен** — у [@BotFather](https://t.me/BotFather).
- **Свой Telegram ID** — у [@userinfobot](https://t.me/userinfobot) →
  в `OWNER_IDS`.
- **Lolzteam токен** — на странице <https://lolz.live/account/api>.
- **Ключ шифрования** — `openssl rand -base64 32`.

### 2. Настроить `.env`

```bash
cp .env.example .env
# заполните TELEGRAM_BOT_TOKEN, OWNER_IDS, ENCRYPTION_KEY
```

### 3. Запустить локально

```bash
go build ./...           # проверка сборки
go run ./cmd/bot         # запуск
```

Миграции применяются автоматически при старте; файл БД создаётся по `DB_PATH`.

### 4. Запустить в Docker

```bash
docker build -t bumpbot .
docker run -d --name bumpbot \
  --env-file .env \
  -e DB_PATH=/data/bot.db \
  -v "$PWD/data:/data" \
  bumpbot
```

## Использование

Откройте чат с ботом и отправьте `/start`. Появится меню:

```
[ 👤 Аккаунты ]  [ 📌 Темы ]
[ 📊 Статистика ] [ 🧾 Логи ]
[ ⚙️ Настройки ]
```

### Добавить аккаунт Lolzteam
👤 Аккаунты → ➕ Добавить → Lolzteam → ввод метки → ввод токена.
Сообщение с токеном автоматически удаляется из чата; сразу выполняется проверка
авторизации.

### Добавить аккаунт Mipped
👤 Аккаунты → ➕ Добавить → Mipped → метка → логин → пароль.
Сообщение с паролем удаляется; выполняется тестовый логин, cookies кэшируются и
переиспользуются (перелогин — только при протухшей сессии).

### Добавить тему
📌 Темы → ➕ Добавить тему → выбрать аккаунт →
- для **Lolz** ввести числовой `thread_id`;
- для **Mipped** вставить URL вида `https://mipped.com/f/threads/slug.12345/`.

Затем интервал (или `-` для значения по умолчанию). Заголовок подтянется
автоматически.

## Как это работает

- **Lolz:** `POST /threads/{id}/bump` для апа, `GET /threads/{id}` для
  статистики, `GET /users/me` для проверки токена. Имена полей JSON
  обрабатываются и с префиксом `thread_`, и без.
- **Mipped:** логин → парсинг свежего `_xfToken` со страницы темы (data-csrf /
  hidden input / `XF.config.csrf`) → `POST /threads/{slug}.{id}/up` с JSON-телом.
  Ответ-403 с текстом «Нужно подождать N…» не считается фатальным: остаток
  времени парсится и `next_bump_at` переносится на `now + остаток + 60 сек`.
- **Планировщик:** тик раз в 30 сек выбирает темы с `next_bump_at <= now`
  (или `NULL`). После апа `next_bump_at = now + interval + rand(jitter)`. Между
  запросами выдерживается `REQUEST_DELAY_MS`.

## Анти-бан / аккуратность

- Реалистичный постоянный User-Agent (десктопный Firefox), никогда не
  `Go-http-client`.
- Джиттер по времени — апы не в HH:00:00.
- Глобальная задержка между запросами.
- Опциональный socks5-прокси на аккаунт.
- При ошибках — backoff, без флуда ретраями.
- Lolzteam — только официальный API. Mipped — только штатная кнопка апа в рамках
  суточного лимита, который контролирует сам сервер.

## Конфигурация

См. [`.env.example`](.env.example). Числовые параметры (интервалы, джиттер,
период статистики, задержка) также редактируются в панели → ⚙️ Настройки и
применяются на лету.

> ⚠️ Секреты в БД зашифрованы ключом `ENCRYPTION_KEY`. Не теряйте и не меняйте
> его — иначе сохранённые токены/пароли/cookies станут нечитаемыми. `.env` и
> `*.db` уже в `.gitignore`.

## Заметки по реализации

- Слой хранилища — интерфейс `Storage`; SQLite-реализацию можно заменить на
  Postgres (`pgx`) без правок остального кода.
- Эндпоинты Lolz и формат `/up` Mipped взяты из проверенного ТЗ дословно.
- Селекторы статистики Mipped — best-effort (`dl.pairs` + локали RU/EN); если
  поле не распарсилось, сохраняется `NULL`, бот не падает.
